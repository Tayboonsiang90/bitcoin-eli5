A short introduction to cryptography

Hash functions

A hash function is a function that accepts an input (plaintext) of any length and outputs a scrambled output (ciphertext) of fixed length.

Try out the example below!

Plaintext
Ciphertext (HEX)

The output of a hash changes drastically when the message is changed.

Even if the message is changed by only a single letter, the hash becomes unrecognizable.

A hash function is computationally difficult to reverse.

Given a hash, it is hard to guess the message that returns the hash. These are known as one-way functions.

Security

Hash functions have 3 important properties.

  1. Preimage resistance: Given a message hash hash(M), it is hard to find the original message M.

  2. Second preimage resistance: Given a message hash hash(M), if it hard to find another different message such that that have the same hash.

  3. Collision resistance: It is hard to find any two different message such that they have the same hash.

Notice how #3: Collision resistance is the weakest property.

Hence, if a hash function is collision resistant, #1 and #2 are assured as they are stronger properties.
